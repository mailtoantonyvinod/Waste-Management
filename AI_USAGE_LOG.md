## Part 2: Reflection Report

### Which AI tools were used and how?
I used **Claude (claude.ai)** throughout this project. I used it primarily as a coding assistant — to generate boilerplate code, explain concepts I was unfamiliar with (like JWT middleware and React Context), and scaffold the project structure. I did not ask Claude to write my reflection or explanation for me.

### Parts generated by AI vs manually coded

**AI-Generated (with my review and understanding):**
- Express server setup and route structure
- SQLite schema and seed script
- JWT authentication middleware
- All React page components (Login, Register, Resident/Worker/Admin dashboards)
- Axios API instance with interceptors
- README documentation

**Written/Modified by me:**
- Zone assignment logic after registration (I added zone validation)
- Demo credential quick-fill buttons on the login page (I added this for evaluators)
- Customised the filter logic in AdminDashboard after understanding it
- This reflection report and AI usage log
- All Git commit messages and commit history

### Did AI help or hinder understanding?

AI **helped** significantly. As a React beginner, I was unfamiliar with concepts like `useContext`, `useEffect`, and component re-rendering. Claude explained these step by step whenever I asked. However, the risk is that code can be generated faster than it can be understood. To counter this, I made it a habit to:
1. Read every generated file before running it.
2. Ask Claude to explain anything I didn't understand.
3. Make small deliberate changes to test my understanding.

### Issues encountered integrating AI output

1. **CORS issue:** The backend initially rejected frontend requests. I had to understand that the `proxy` field in `package.json` handles this in development — Claude explained this when I asked.
2. **SQLite path issue:** The database file was being created in the wrong directory. I fixed this by understanding `path.join(__dirname, ...)`.
3. **JWT expiry:** I initially set it too short (1h) and got logged out frequently. I changed it to `7d` and understood the tradeoff with security.

### What did I learn from debugging AI-generated code?

Debugging AI code is no different from debugging any code — the errors still happen, and you still need to understand the system to fix them. The biggest learning was that AI-generated code can have hidden assumptions (e.g., it assumed the `zone` field is always populated for residents, which wasn't true for admin users). I traced the error, found the assumption, and added a null check. This kind of debugging actually deepened my understanding more than just reading the code would have.

### Overall Assessment
Claude accelerated my development speed by 3–4x for this project. The value is not in replacing my understanding, but in removing the friction of boilerplate so I can focus on understanding the concepts that matter. The code I submitted is code I understand and can defend.
