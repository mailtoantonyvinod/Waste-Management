# SE ZG503 Submission Checklist

Use this checklist before final LMS/GitHub submission.

## 1) Code + Runtime

- [ ] Backend dependencies install successfully (`cd backend && npm install`)
- [ ] Frontend dependencies install successfully (`cd frontend && npm install`)
- [ ] Database seeded (`cd backend && npm run seed`)
- [ ] Backend runs (`cd backend && npm run dev`)
- [ ] Frontend runs (`cd frontend && npm start`)
- [ ] Login works for admin/worker/resident demo users
- [ ] Core flows work end-to-end (create request, update status, view stats)

## 2) Assignment Feature Mapping

- [ ] React frontend with modular components and routing
- [ ] Backend APIs implemented with validation and role checks
- [ ] Persistent database with schema documented
- [ ] Frontend-backend integration demonstrated
- [ ] Problem statement and design rationale documented

## 3) Documentation

- [ ] `README.md` includes setup, architecture, API docs, schema, assumptions
- [ ] `AI_USAGE_LOG.md` has real student name/ID/date (no placeholders)
- [ ] Architecture diagram/wireframe added (if required by instructor)
- [ ] API documentation format ready (README/Postman/Swagger)

## 4) Git + Repository

- [ ] Repository is initialized as git repo
- [ ] Commit history is clean and meaningful (not one massive commit)
- [ ] `.env` is not committed; only `.env.example` is tracked
- [ ] Repository visibility is set to **Public**

## 5) Final Submission Package

- [ ] GitHub repo link submitted
- [ ] Demo video uploaded to Google Drive with evaluator access
- [ ] LMS upload includes required documents
- [ ] AI policy and academic honesty requirements reviewed
